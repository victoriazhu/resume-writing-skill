# 多岗位简历写作 · 动词与模板指导

> 配合 resume-writing skill v2.0+ 使用。每个岗位列出：核心动词优先级、典型量化指标、需避免的通用表述、以及一条"弱→强"改写示例。

---

## 后端/服务端工程师

**核心动词优先级**：设计 > 实现 > 重构 > 优化 > 搭建 > 治理 > 封装 > 抽象 > 迁移

| 错误写法 | 改写方向 |
|---------|---------|
| 负责XX系统的开发和维护 | 设计并实现了日均QPS 5万的订单查询服务，P99延迟从500ms降至80ms |
| 参与XX项目开发 | 作为核心开发者重构了消息队列中间件，吞吐量提升3倍，单机QPS从2k提升至8k |
| 优化系统性能 | 通过引入多级缓存+读写分离架构，将核心链路的TP99从1.2s降至120ms |

**典型量化指标**：QPS / TP50/TP99 / SLA（可用性百分比） / 故障恢复时间MTTR / 部署频率 / 代码覆盖率 / 单元测试数量 / 服务数 / 数据量级

**避免的词汇**：负责、参与、协助、支持、进行（属于弱动词，全部需要替换）

## AI 产品经理

**核心动词优先级**：定义 > 设计 > 推动 > 落地 > 协调 > 分析 > 评估

| 错误写法 | 改写方向 |
|---------|---------|
| 负责AI产品的需求分析 | 定义并推动了AI对话产品的需求规划，上线后DAU突破50万，用户留存率提升15% |
| 参与模型优化 | 主导模型评估体系搭建，推动准确率从82%提升至91%，召回率提升8pp |
| 负责项目管理 | 协调算法、工程、设计三方团队，推动AI审核系统3个月完成从0到1上线 |

**典型量化指标**：DAU/MAU / 模型准确率/召回率/F1 / 用户留存率 / 任务完成率 / 人工介入率降低 / ROI

## C 端产品运营

**核心动词优先级**：策划 > 执行 > 分析 > 优化 > 拉动 > 提升 > 撬动

| 错误写法 | 改写方向 |
|---------|---------|
| 负责用户增长 | 策划并执行了"分享得福利"裂变活动，单月拉动新增用户12万，CAC降低35% |
| 做活动运营 | 设计618大促活动玩法，活动期间GMV达800万，同比提升40%，活动参与率35% |
| 分析数据 | 搭建用户分层运营体系（RFM模型），高价值用户复购率从22%提升至38% |

**典型量化指标**：DAU/MAU / 留存率（次日/7日/30日） / 活动参与率 / 转化率（各漏斗节点） / CAC / ROI / GMV
